[面经]腾讯PCG后台开发二面

2020-04-01

写在前面

昨天下午三点电话面完一面，手撕从七点到九点，晚上十点左右状态就更新到复试了。然后今天下午两三点，二面的面试官就打电话来和我约时间了。他问我今明两天有空吗？我脑子一热就选了今天;_;)然后晚上七点就开始了。

问题

选择题

上来两道选择题：

  • 线程共享以下哪些资源：
    A. 栈
    B. 数据段
    C. 寄存器
    D. TID

ez，B

  • 下列操作中，多线程环境需要加锁的是？
    A. x=y
    B. x++
    C. x=x+1
    D. x=1

这题我只知道从原子操作角度入手，但是我只果断排除了 D。

我不知道 y 是不是临界区的变量，所以我不敢直接排除，问面试官结果面试官也说暂且当它是局部变量，所以我就没选。。。

应该直接用原子操作原则 ABC 全选上的。x++我记成原子操作了，我还是基础知识不到位，最后就单选了一个 C。

判断题

然后面试官问了几道判断题，和动态链接库有关的：

  • 动态链接库比静态链接库快，对吗？

    • 前面两三题都是动态&静态链接库比快慢，不过比了哪些方面我忘了=、=

  • 动态链接库可以显式加载/运行时加载，对吗？

问答题

操作系统

  • 死锁
  • 死锁的检测
  • 中断处理
    • 面试官在这里问我：如果中断的处理函数太久了那怎么办？我也没 get 到这里的点。然后面试官问我，有没有接触过响应和处理分离式的中断处理，我到这里才有点印象，我记得这个是哪个版本的 linux 内核的处理方式？2.x 的？
    • 查了下，是 tasklet 的上半部/下半部机制 555

计网

  • 我在浏览器里输入一个网址，比如 www.qq.com，浏览器从打开这个网址到返回页面为止做了些啥。
  • 讲讲三次握手，为啥不能是两次？
    • 拓展：那讲讲四次挥手，为啥这里就要四次而不是三次？
  • 你刚才讲到 HTTP 和 HTTPS，有了解过区别吗？
    • 稍微讲讲 HTTPS 的连接呗
    • 那 HTTPS 里加密对称密钥的过程是怎么做的？（问非对称加密过程）

数据结构

  • 设计数据结构：
    • 设计一个数据结构供简化的 CPU 调度使用，有以下两个调度函数：
      • ADD(time,function)
        • 往任务队列里插入一个计划：在 time 秒后执行 function 函数
      • RUN()
        • CPU 每秒都会执行这个 RUN()函数，运行当前秒所需要执行的函数
    • 我用的方案是循环数组+开链表/红黑树（效仿 hashMap）
      • 面试官和我探讨了一会儿这个问题，他就提出了不如只用链表，然后因为 ADD 的插入过程可以做排序，RUN 只需要遍历即可。这个面试官确实很好交流哈哈哈。

问项目

问了我 C/C++的项目，其实这是个偏硬件方向的项目，问了我技术选型的问题：为什么选这个系统呢，然后问了我有没有什么印象深刻的困难或者经历。

其他讨论

问了下我可以实习的时间，我说我的时间还是比较宽松的，三个月肯定有。然后就大家收工互道再见了。
