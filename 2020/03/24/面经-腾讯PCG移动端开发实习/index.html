[面经]腾讯PCG移动端开发实习

2020-03-24

写在前面

网申投简历的时候，看到移动端开发录取比是 1/4，PC 端开发录取比是 1/7，然后就投了移动端开发 Orz，现在想想确实是浪费机会了，还是投自己最擅长的领域比较好，出门就学到了东西也好。然后被捞起来过了四五天才打电话来通知面试，我差点以为我被晾了。第一意向事业群是 WXG，结果还是被 PCG 捞起来的。

面试

电话面试，约的 0323 晚上八点，面了 34 分钟。

问技术

OS

开门就是 OS，按顺序问了以下问题：

  • 进程和线程了解吗？讲讲区别。
  • 进程间通信有什么方式？
  • 你刚刚说你用过共享内存对吧，那讲讲怎么用的。
  • 死锁是什么
  • 怎么检测解决死锁？
  • 进程间的同步机制
  • 情景题：我在一个可重入的函数中有一个变量，访问这个变量要加锁吗？
    • 追问：那这个变量保存在哪里？
  • 情景题：那我读一个临界区的资源时不加锁，写的时候加锁，可以吗？
    • 我回答的是可以，但我画蛇添足了一段：按规范来说读写都是要加锁的
    • 追问：为什么？

计网

  • 情景题：我在浏览器中输入一个网址，从我访问这个链接到返回页面的全过程
  • 讲讲 HTTP 的状态码
    • 这里我没有复习到，所以只讲了我之前接触过的 301/302/404/502。
  • （承接上个问题）那你知道正常的返回码是多少吗？
    • 这个就是我没复习到的，我只回答了：我记得是 2 开头的状态码。
    • 面完查了一下，确实是 2 开头的，不过具体的代码都有不同的含义。
  • 讲讲 HTTP1.0 和 HTTP1.1 的差别？
    • 我不会，只根据印象讲了一个 keep-alive 变成默认值。

C++/数据结构/Java

面试官问了下我：你之前说你主要用 C++还是 Java 来着？我说我用 C++多，然后面试官就开始问 C++了，但是问着问着又混进来 Java 的一些数据结构。

  • 虚函数怎么实现的
  • 用过 hashMap 吗？底层怎么实现的？
  • hash 冲突怎么办？
  • 如果要加长怎么办？
    • rehash，这里我答了 rehash 的流程，不过感觉面试官不是太满意。我也没敢问有什么问题。
  • Java 里面的 linkedHashMap 用过吗？
    • 面完出来查一下才知道我用过，我服了我自己了。
  • Map 的底层实现是什么？
    • 这里应该说的是 C++的 Map，我还以为是承着上一问在问 Java，想了一会儿才转回到 C++上来。
  • 设计数据结构：
    • 性质 1：是个集合，元素唯一。
    • 性质 2：能够按我输入的顺序取出。
    • 我面的时候说的是用 Set+List，List 上记录指针指向 Set 上的节点。我说完之后面试官问我：是不是有数据冗余？我就说我在 List 上只记录指针不记录数据 blabla，感觉面试官是勉强通过了。
    • 应该用一个 Set 就行，key=插入的元素，value=一对指针指向上一个/下一个节点。

问项目

  • 介绍一下你的项目
  • 问了一下项目中的具体实现
  • 讲讲另一个项目

结果

一面挂，我觉得首先是有些东西我还是没有掌握好，其次是我有点话多，还是要更精确地回答问题，不是说的多就好。第一次面试就这么没了，不过肯定会继续的。
