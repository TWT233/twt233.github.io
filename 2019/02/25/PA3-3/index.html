分页这部分我也写了比较久，分页处理的小问题比较多，而且脑子比较糊的我犯了蛮多错误，分页这个机制也很有意思，实现了多进程管理。

PA3-3

PA3-2中实现的segment将vaddr转换成laddr，laddr经过3-3的分页处理就会转换成物理地址。

首先要实现的是GDTR，这是page_translate的起点。之后的填表操作就交给已经实现好的mm_malloc在loader()函数中进行了。

接下来要实现的是lgdt指令，这个指令我最开始理解有问题，m16&32这个操作数类型其实是一个指针，并不是立即数，可以直接用instr_helper中的rm和l类型来做这个。

  • 注意这个并不需要进行operand_read，直接使用opr_src.addr进行读写即可。具体为什么可以看框架中的decode_operand的实现和实际的lgdt指令样式（反编译一下kernel就能找到）

除了上面那个操作数类型的问题以外，我沙雕的地方还在mov creg的一套指令中把我坑惨了。我用俺寻思法在PA3-2里写完了mov creg一套指令，不知道为啥当时没出问题，在3-3里才出问题。

当然这是正确版本，当然这个是我抄上面的r2rm的，但是事实证明我抄作业都没抄好。

本着俺寻思法的精髓，我寻思了下：r2rm对应&opr_src,&opr_dest这样的传参顺序，那么不就说明第二个参数位置要放r型操作数嘛，所以r2c应该写成：len += modrm_r_rm(eip + 1, &opr_src, &opr_dest);，我一拍脑子就这么写了上去。

结局就是我debug了半晚上，俺寻思法有的时候还是害人啊。
